Anotações para a Prova 01:

Processo: um container onde estão todos os passos e informações necessárias para executar um determinado programa. Contem valores do pc, das variáves, dos registradores...

Gerenciamento de processos: 
A) -> um programa em execução é um processo, uma unidade de trabalho no sistema.
B) -> Recurso são disponibilizados na criação ou durante a execução, tudo que for reutilizado, precisa ser liberado, um programa é uma entidade passiva e um processo, ativa.
Monothread x Multithread : 
Única thread = um único contador de programa - Múltiplos threads = um contador de programa por thread.
Funções: 
A) -> Scheduling de processos e threads da CPU,
B) -> Criação e exclusão de processos do usuário e de sistema ,
C) -> Suspensão e retomada de processos,
D) -> Fornecimento de mecanismos de sincronização de processos,
E) -> Fornecimento de mecanismos de comunicação entre processos.

Gerenciamento de Memória: É onde é armazenado os dados e instruções, onde é feito o processo do ciclo de Busca, de Von Neumann.
Funções: 
A) -> Controlar que partes da memória estão em uso concorrente e quem está usando,
B) -> Decidir que processos e dados devem ser colocados dentro ou fora da memória,
C) -> Alocar e desalocar espaço na memória conforme necessário.

Sistema de Arquivos : Armazena arquivos em diretórios, arquivos são feitos pelo seu criado em diferentes extensões, seja .txt ou alfanuméricos ou binários, normalmente representam dados ou programas.
Gerenciamento de Armazenamento: SO provê um armazenamento lógico e uniforme para armazenar as infos..
Propriedades: 
A) -> velocidade de acesso, taxa de transferência de dados, organizados em diretório, controle de Acesso.
Gerenciamento do Sistema de Arquivos: um SO da a possibilidade de um computador armazenar vários tipos de arquivos, logo é necessário.
Propriedades: 
A)-> criar e apagar arquivos, criar e apagar diretórios para organizar os arquivos, suportar a manipulação de arquivos no diretório.
B) -> Mapear arquivos para memória secundária, criar cópias de arquivos em mídia de armazenamento estáveis e não voláteis.
Gerenciamento de Armazenamento em Massa: Discos são geralmente utilizados para armazenar dados que não cabem na 
memória principal ou que devem ser mantidos por um “longo” período de 
tempo.
Propriedades:
A) -> Gerenciamento de espaço livre, alocação de espaço de armazenamento,
B) -> Scheduling de alocação de disco.
Armazenamento terciário:  inclui armazenamento ótico, fitas magnéticas e precisa ser gerenciado.
Armazenamento em Cache: armazena a medida que é utilizado, um ambiente com múltiplos processadores precisa prover coerência de 
cache em hardware para garantir que todos as CPUs tenham o valor mais 
recente em seus caches.
Armazenamento IO: interface uniforme, logo é um interface geral para o Driver, Bufferização (armazenamento de dados temporários enquanto dados estão 
sendo transferidos)

Proteção e Segurança: 
 Identidades dos usuários (IDs) incluem nome e número associado, um 
por usuário, ID do usuário é então associado com todos os arquivos, processos 
deste usuário para determinar seu controle de acesso 
○ Identificador de grupo (group ID) permite que um conjunto de usuários 
serem definidos e controles de gestão. 
○ Mecanismo de mudança de privilégios permite ao usuário alterar para 
um ID com mais direitos
